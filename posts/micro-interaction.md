# 微交互设计：打造更强反馈

微交互是用户与界面之间最细微却最频繁的接触点。一次按钮点击、一条加载动画、一个表单校验提示——它们共同决定了产品是否「好用」。

## 什么是微交互

微交互通常包含四个阶段：

1. **触发** — 用户操作或系统事件
2. **规则** — 决定发生什么
3. **反馈** — 视觉、听觉或触觉响应
4. **循环** — 重复或结束条件

## 设计原则

### 即时反馈

用户操作后 100ms 内应有可见响应。延迟超过 400ms 会让用户怀疑是否操作成功。

### 有意义的活动指示

加载状态应传达进度感，而非无限旋转。骨架屏、进度条、分步动画都是更好的选择。

### 情感化细节

适度的弹性动画、颜色变化能让界面更有「生命感」，但切忌过度——动效应服务于功能，而非炫技。

## 常见场景

| 场景 | 建议 |
|------|------|
| 按钮点击 | 轻微 scale + 阴影变化 |
| 表单提交 | 禁用按钮 + loading 态 |
| 成功/失败 | 图标 + 颜色 + 简短文案 |
| 列表加载 | 骨架屏优于 spinner |

## 结语

好的微交互让用户感觉不到设计的存在，只觉得一切自然流畅。从最关键的用户路径开始，逐个打磨这些细节。
